# Objective

Enable Prometheus-to-backend scrape authentication in the production config and document the deployment steps. Currently the `authorization` block in `prometheus.prod.yml` is commented out, meaning Prometheus either fails to scrape or scrapes unauthenticated. Also address the `web.yml` envsubst gap.

# Findings Addressed

- **NEW-002** (Medium): Prometheus-to-backend scrape auth commented out in prod config

# Implementation Plan

1. Uncomment the `authorization` block in `prometheus.prod.yml` with `${PROMETHEUS_BEARER_TOKEN}` variable
2. Document in monitoring README the required env vars and envsubst step
3. Add pre-deploy checklist section covering `METRICS_API_KEY`, `PROMETHEUS_BEARER_TOKEN`, `web.yml` processing
4. Verify dev compose still works without the bearer token (dev config is separate)
