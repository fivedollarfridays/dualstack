# Objective

Fix R5 NEW-007: `ContentSizeLimitMiddleware` in `security_headers.py` only checks the `Content-Length` header. Requests using chunked transfer encoding omit this header, bypassing the 1MB body size limit entirely. Add streaming body size enforcement that counts bytes as they arrive.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/core/security_headers.py` | Add streaming byte counter to `ContentSizeLimitMiddleware` that reads the body incrementally and rejects requests exceeding `MAX_BODY_SIZE` regardless of `Content-Length` presence |
| `backend/tests/core/test_security_headers.py` | Add tests for chunked transfer encoding bypass, streaming large body rejection, and normal requests still passing |

# Implementation Plan

1. Write failing tests:
   - Test that a request without `Content-Length` but with a body exceeding 1MB is rejected (413)
   - Test that a chunked transfer request under 1MB passes
   - Test that existing `Content-Length` check still works
2. Modify `ContentSizeLimitMiddleware.__call__`:
   - Keep the existing fast-path `Content-Length` header check
   - Add a streaming body wrapper that counts bytes read from `receive()`
   - If accumulated bytes exceed `MAX_BODY_SIZE`, return 413 immediately
3. Verify all existing tests still pass

# Acceptance Criteria

- [x] Requests with chunked transfer encoding exceeding 1MB return 413
- [x] Requests with chunked transfer encoding under 1MB pass through normally
- [x] Existing Content-Length header check still works (no regression)
- [x] Streaming body counter does not buffer the entire body in memory
- [x] All existing middleware tests pass
- [x] New tests cover: chunked bypass, streaming limit, boundary cases
