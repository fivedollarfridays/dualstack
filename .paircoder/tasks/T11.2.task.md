# Objective

Fix R5 NEW-008: Backend `url_validation.py` does not check for embedded credentials in URLs (e.g., `http://attacker:password@localhost:3000`). The frontend validator already rejects these (`parsed.username || parsed.password`), but the backend omits this check. Add parity with the frontend validator.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/core/url_validation.py` | Add `parsed.username` / `parsed.password` rejection after URL parsing |
| `backend/tests/core/test_url_validation.py` | Add tests for URLs with embedded credentials (username only, password only, both) |

# Implementation Plan

1. Write failing tests:
   - `http://attacker@localhost:3000` — rejected
   - `http://user:pass@example.com` — rejected
   - `http://:password@example.com` — rejected
   - Normal URLs without credentials — still accepted
2. Add credential check in `url_validation.py`:
   - After `urlparse()`, check `parsed.username` or `parsed.password`
   - Return `False` / raise validation error if present
3. Verify frontend parity (both validators now reject embedded credentials)

# Acceptance Criteria

- [x] URLs with username component are rejected
- [x] URLs with password component are rejected
- [x] URLs with both username and password are rejected
- [x] Normal URLs without credentials continue to work
- [x] Backend validator matches frontend credential check behavior
- [x] All existing URL validation tests pass (no regression)
