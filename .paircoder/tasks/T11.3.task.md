# Objective

Fix R5 NEW-011: `DevAuthProvider` component uses `console.warn()` only to signal dev-mode auth is active. There is no build-time guard preventing this component from being included in a production bundle. Add a runtime check that throws an error if `DevAuthProvider` is rendered in a production environment.

# Files to Update

| File | Change |
|------|--------|
| `frontend/src/components/auth/dev-auth-provider.tsx` | Add runtime guard that throws if `process.env.NODE_ENV === 'production'` |
| `frontend/src/components/auth/dev-auth-provider.test.tsx` | Add test verifying the production guard throws |

# Implementation Plan

1. Write failing test:
   - Mock `process.env.NODE_ENV` to `'production'`
   - Verify rendering `DevAuthProvider` throws an error
   - Verify rendering in `'development'` still works with console.warn
2. Add guard at the top of `DevAuthProvider` component body:
   - `if (process.env.NODE_ENV === 'production') throw new Error('...')`
   - Keep existing `console.warn` for dev mode
3. Verify all existing auth tests pass

# Acceptance Criteria

- [x] `DevAuthProvider` throws an error when `NODE_ENV=production`
- [x] `DevAuthProvider` works normally in development mode
- [x] Error message clearly states dev auth is not for production
- [x] Existing console.warn behavior preserved in dev mode
- [x] All existing auth and provider tests pass
