# Objective

No seed data script exists for fresh development environments. Developers setting up the project locally must manually create sample records to test functionality. This task creates an idempotent seed script that populates the database with realistic sample data for development and testing.

The script should:
- Create sample records for all major tables (items, etc.)
- Be idempotent: safe to run multiple times without creating duplicates
- Use the existing SQLAlchemy models and session management from `database.py`
- Provide clear console output indicating what was created vs. skipped

# Files to Update

- `backend/scripts/seed.py` -- **New file**: Main seed script
- `backend/app/items/models.py` -- Reference only (read existing models to understand schema)
- `backend/app/core/database.py` -- Reference only (use existing session/engine)
- `docs/ARCHITECTURE.md` or `README.md` -- Document seed script usage
- `backend/scripts/__init__.py` -- Create if `scripts/` directory does not exist

# Implementation Plan

## Phase 1: Understand existing models

1. Read `backend/app/items/models.py` to understand the Item model fields and constraints
2. Read any other model files to identify additional tables that need seed data
3. Read `backend/app/core/database.py` to understand session creation and engine setup
4. Identify unique constraints and required fields for each model

## Phase 2: Write seed script (TDD)

5. Write tests for the seed logic:
   - Test that seed creates expected records in an empty database
   - Test that seed is idempotent (running twice produces the same result)
   - Test that seed handles existing records gracefully (no duplicates, no errors)
6. Create `backend/scripts/seed.py` with:
   - A `seed_items()` function that creates sample items
   - Additional `seed_<table>()` functions for other models as needed
   - A `main()` function that orchestrates all seed functions
   - Idempotency: check for existing records before inserting (use unique fields)
   - Console output: print what was created ("Created item: ...") vs skipped ("Item already exists: ...")
7. Make the script runnable via `python scripts/seed.py` from the `backend/` directory

## Phase 3: Sample data design

8. Create 5-10 sample items with realistic, varied data:
   - Different names, descriptions, and field values
   - Cover edge cases (long names, special characters if applicable)
   - Use deterministic data (not random) so results are reproducible

## Phase 4: Documentation

9. Add usage instructions to README or docs:
   ```
   cd backend
   python scripts/seed.py
   ```
10. Note that the script requires `DATABASE_URL` to be set (or uses the default local database)
11. Document that the script is idempotent and safe to re-run
