# Objective

The `/api/v1/billing/portal` endpoint at `backend/app/billing/routes.py:34-52` currently raises a 501 with a message explaining that it needs a user->customer mapping. Now that T13.1 provides the `users` table with `stripe_customer_id`, implement this endpoint so authenticated users can access the Stripe Billing Portal to manage their subscription.

# Background

- The portal route stub exists at `backend/app/billing/routes.py:34-52` and already has auth dependency (`get_current_user_id`), rate limiting, and a docstring explaining what it should do.
- `backend/app/billing/service.py:create_portal_session` already wraps `stripe.billing_portal.Session.create` and accepts a `customer_id` and `return_url`.
- The `PortalRequest` schema in `backend/app/billing/schemas.py:28-35` accepts a `return_url` with URL validation.
- The Users service from T13.1 provides `get_user_by_clerk_id(db, clerk_user_id)` which returns the user record (or None).

# Implementation Plan

## 1. Update the portal route (`backend/app/billing/routes.py`)

Replace the 501 stub in `create_portal`:

1. Accept `data: PortalRequest` body parameter (already has the schema)
2. Add `db: AsyncSession = Depends(get_db)` dependency
3. Look up the user by `clerk_user_id` using `users_service.get_user_by_clerk_id(db, user_id)`
4. If user not found or `stripe_customer_id` is None, return 404 with clear message ("No billing account found. Please subscribe first.")
5. Call `service.create_portal_session(user.stripe_customer_id, data.return_url)`
6. Return `{"url": portal_url}`
7. Add audit event for `billing.portal_created`

## 2. Add necessary imports to routes

- Import `AsyncSession`, `Depends`, `get_db`
- Import `users` service module (or specific function)
- Import `PortalRequest` (already imported? verify)

## 3. Write tests

- `backend/tests/test_billing/test_portal_route.py`
  - Test: authenticated user with `stripe_customer_id` gets portal URL
  - Test: authenticated user without `stripe_customer_id` gets 404
  - Test: user not found in database gets 404
  - Test: Stripe API error returns 502/503
  - Test: unauthenticated request returns 401

Mock `stripe.billing_portal.Session.create` in all tests. Use the test database with a seeded user row.
