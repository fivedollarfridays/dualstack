# Objective

The dashboard page at `frontend/src/app/(dashboard)/dashboard/page.tsx` currently hardcodes "Free" as the subscription plan (line 27). Replace this with real subscription data fetched from the backend, showing the user's actual plan name, subscription status, and providing links to manage billing.

# Background

- The dashboard page is at `frontend/src/app/(dashboard)/dashboard/page.tsx` and uses the `Card` component and `useItems` hook.
- The frontend API layer is at `frontend/src/lib/api/` with `billing.ts`, `items.ts`, and `shared.ts` providing fetch helpers.
- Custom hooks live in `frontend/src/hooks/` (e.g., `use-items.ts`).
- The billing API client at `frontend/src/lib/api/billing.ts` has `createCheckout` but no subscription/user data fetching.
- Auth tokens are available via the Clerk auth context at `frontend/src/contexts/auth-context.tsx`.
- T13.1 creates the backend Users model. A backend endpoint to fetch the current user's subscription info will need to be created (or confirmed to exist) as part of this task's implementation.

# Implementation Plan

## 1. Add backend endpoint for user subscription info

Create a minimal endpoint (if not already present from prior tasks):

`GET /api/v1/users/me` -- returns the authenticated user's subscription info:
```json
{
  "clerk_user_id": "user_abc",
  "subscription_plan": "pro",
  "subscription_status": "active"
}
```

This may go in a new `backend/app/users/routes.py` or be added to the billing routes. If the user record doesn't exist, return `{"subscription_plan": "free", "subscription_status": "none"}` as a default.

## 2. Add frontend API function (`frontend/src/lib/api/billing.ts`)

Add a function to fetch subscription data:

```typescript
export async function getSubscription(token: string): Promise<SubscriptionInfo> {
  const res = await fetch(`${API_URL}/api/v1/users/me`, {
    headers: authHeaders(token),
  });
  return handleResponse<SubscriptionInfo>(res);
}
```

Define the `SubscriptionInfo` type:
```typescript
interface SubscriptionInfo {
  subscription_plan: string;
  subscription_status: string;
}
```

## 3. Create `useSubscription` hook (`frontend/src/hooks/use-subscription.ts`)

A custom hook that fetches subscription data on mount:

```typescript
export function useSubscription() {
  // Returns { plan, status, isLoading, error }
  // Uses the auth token from context
  // Fetches from getSubscription API function
}
```

Handle loading, error, and fallback states:
- While loading: show skeleton/spinner
- On error: fall back to "Free" with no error shown to user
- On success: display actual plan and status

## 4. Update dashboard page (`frontend/src/app/(dashboard)/dashboard/page.tsx`)

Replace the hardcoded "Free" with data from `useSubscription`:

- Show plan name (capitalize first letter: "free" -> "Free", "pro" -> "Pro")
- Show subscription status as a badge or subtitle (e.g., "Active", "Past Due", "Canceled")
- Add a "Manage Subscription" link that navigates to the billing page (or opens portal)
- Add an "Upgrade" link for free-tier users
- Graceful fallback: if subscription data fails to load, show "Free" as default
