# Objective

Playwright E2E tests exist in the project but are not executed in CI. Add a GitHub Actions job that installs Playwright browsers, starts the application, runs the E2E test suite, and uploads test artifacts (screenshots, videos, traces) on failure. E2E failures must block PR merges.

# Files

| File | Action |
|------|--------|
| `.github/workflows/ci.yml` | Update -- add E2E test job |
| `playwright.config.ts` | Verify -- ensure CI-compatible settings (headless, retries, reporter) |

# Implementation Plan

1. **Review existing `playwright.config.ts`** for CI readiness:
   - Confirm `headless: true` is set (or defaults to headless in CI).
   - Check for a `webServer` config that starts the app automatically.
   - Verify reporter settings include a CI-friendly reporter (e.g., `html`, `list`, or `github`).
   - Confirm retry count is reasonable for CI (e.g., `retries: process.env.CI ? 2 : 0`).
   - Check `outputDir` for test artifacts and `use.trace` / `use.video` / `use.screenshot` settings.
2. **Add an E2E test job to `.github/workflows/ci.yml`**:
   - **Job name**: `e2e-tests`
   - **Runs on**: `ubuntu-latest`
   - **Steps**:
     a. Checkout code
     b. Setup Node.js (match project version)
     c. Install dependencies (`npm ci` or equivalent)
     d. Install Playwright browsers (`npx playwright install --with-deps chromium` -- install only needed browsers to save CI time)
     e. Build the application (if needed for E2E)
     f. Run Playwright tests (`npx playwright test`)
     g. Upload test artifacts on failure:
        - Upload `playwright-report/` directory
        - Upload `test-results/` directory (screenshots, videos, traces)
        - Use `actions/upload-artifact@v4` with `if: failure()` condition
   - **Environment variables**: Set `CI=true` and any required env vars.
   - **Timeout**: Set a reasonable job timeout (e.g., 15 minutes) to prevent hanging.
3. **Ensure E2E job blocks merges**:
   - The E2E job should be part of the required status checks.
   - If the CI workflow uses a matrix or multiple jobs, ensure E2E is listed in branch protection rules (document this).
4. **Update `playwright.config.ts` if needed**:
   - Add CI-specific settings if missing.
   - Ensure `webServer` starts the dev or preview server.

# Acceptance Criteria

- [x] E2E test job exists in `.github/workflows/ci.yml`
- [x] Playwright browsers are installed in CI (with system dependencies)
- [x] E2E tests run automatically on every PR
- [x] E2E test failures block PR merge (job is required status check)
- [x] Test artifacts (screenshots, videos, traces) are uploaded on failure
- [x] Job has a reasonable timeout to prevent indefinite hanging
- [x] `playwright.config.ts` has CI-compatible settings (headless, retries, reporter)

# Verification

- Push a branch with a passing E2E test and confirm the CI job passes
- Push a branch with a deliberately failing E2E test and confirm:
  - The CI job fails
  - Test artifacts are uploaded and downloadable
  - The PR is blocked from merging (if branch protection is configured)
- Review the CI job logs for clean output (no warnings about missing browsers or dependencies)
- Verify the job completes within the timeout window
