# Objective

The frontend Docker image is not built or verified in CI, meaning Dockerfile regressions (broken builds, missing dependencies, misconfigured stages) are only caught during deployment. Add a CI job that builds the frontend Docker image on every PR to catch build failures early.

# Files

| File | Action |
|------|--------|
| `.github/workflows/ci.yml` | Update -- add frontend Docker build job |
| `frontend/Dockerfile` | Verify -- confirm it exists and builds correctly |

# Implementation Plan

1. **Verify `frontend/Dockerfile` exists** and can build locally:
   - Run `docker build -t dualstack-frontend-test ./frontend` locally to confirm it builds.
   - Note any build args or secrets required.
2. **Add a Docker build verification job to `.github/workflows/ci.yml`**:
   - **Job name**: `docker-build-frontend`
   - **Runs on**: `ubuntu-latest`
   - **Steps**:
     a. Checkout code
     b. Set up Docker Buildx (`docker/setup-buildx-action@v3`)
     c. Build the frontend Docker image:
        - Use `docker/build-push-action@v5` with `push: false` (build only, no push)
        - Context: `./frontend`
        - File: `./frontend/Dockerfile`
        - Tags: `dualstack-frontend:ci-${{ github.sha }}`
        - Cache: Use GitHub Actions cache (`cache-from` / `cache-to`) for faster builds
     d. Optionally run a basic container smoke test (start container, check health endpoint)
   - **Build args**: Pass any required build-time arguments (e.g., `NEXT_PUBLIC_*` env vars with dummy values for CI).
   - **Timeout**: Set a reasonable timeout (e.g., 10 minutes).
3. **Ensure the job blocks merges**:
   - The Docker build job should be a required status check so that Dockerfile regressions block PRs.

# Acceptance Criteria

- [x] Frontend Docker build job exists in `.github/workflows/ci.yml`
- [x] The job builds the frontend Docker image on every PR
- [x] Build failures block PR merge (job is a required status check)
- [x] Docker layer caching is configured for faster CI builds
- [x] The job does not push images (build verification only)
- [x] Job has a reasonable timeout

# Verification

- Push a branch with no Dockerfile changes and confirm the Docker build job passes
- Introduce a deliberate Dockerfile error (e.g., invalid `COPY` source) and confirm the job fails
- Check CI job duration to ensure caching is effective on subsequent runs
- Verify the job does not attempt to push to any container registry
