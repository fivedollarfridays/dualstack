# Objective

No single-command dev setup exists. Developers must manually start the backend, frontend, and monitoring stack in separate terminals, each with different commands (`uvicorn`, `npm run dev`, `docker-compose` with compose file overrides). Create a root-level Makefile that unifies all common development workflows into simple, discoverable targets. Update the README quickstart to reference the Makefile.

# Files

- `Makefile` (new) -- root-level Makefile with all dev workflow targets
- `README.md` (update) -- replace multi-step quickstart with `make setup && make dev`

# Implementation Plan

1. Create `Makefile` at project root with the following targets:
   - `make setup` -- install backend deps (`pip install -r requirements.txt` + dev deps), install frontend deps (`npm install`), copy `.env.example` to `.env` / `.env.local` if not already present (never overwrite existing env files)
   - `make dev` -- start backend (`uvicorn` with `--reload`), frontend (`npm run dev`), and monitoring stack (`docker-compose` with dev overrides) using background processes or `docker-compose up`
   - `make test` -- run backend tests (`pytest`) and frontend tests (`npm test`) sequentially, exit non-zero if either fails
   - `make build` -- build Docker images for backend and frontend via `docker-compose build`
   - `make clean` -- stop all running services, remove containers, remove `__pycache__`, remove `node_modules/.cache`
   - `make help` -- display available targets with descriptions (default target)
2. Ensure each target uses appropriate working directories (`cd backend && ...`, `cd frontend && ...`)
3. Use `.PHONY` for all targets
4. Update README.md quickstart section to reference the Makefile targets

# Acceptance Criteria

- [x] `make setup` installs backend and frontend dependencies and creates `.env` files from `.env.example` templates without overwriting existing env files
- [x] `make dev` starts backend, frontend, and monitoring stack services
- [x] `make test` runs both backend (`pytest`) and frontend (`npm test`) test suites, exiting non-zero on any failure
- [x] `make build` builds Docker images via docker-compose
- [x] `make clean` stops services and removes build artifacts
- [x] `make help` (or bare `make`) displays available targets with descriptions
- [x] README.md quickstart section updated to reference Makefile targets
- [x] All Makefile targets use tabs for indentation (not spaces)
