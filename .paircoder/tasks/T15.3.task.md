# Objective

Logs are container-local only -- a SOC2 CC7.1 gap. When containers restart or are removed, logs are lost. There is no centralized view of logs across the backend, frontend, and monitoring services. Add a Fluent Bit sidecar to the Docker Compose stack that collects logs from all containers and forwards them to a configurable output. Use Loki as the default output for local development (pairs with the existing Grafana instance) and document how to swap in production backends (Datadog, CloudWatch, Elasticsearch).

# Files

- `monitoring/fluent-bit/` (new directory) -- Fluent Bit configuration
  - `monitoring/fluent-bit/fluent-bit.conf` -- main Fluent Bit config (inputs, filters, outputs)
  - `monitoring/fluent-bit/parsers.conf` -- log parsers (JSON for structured logs, regex for others)
- `monitoring/docker-compose.yml` (update) -- add fluent-bit and loki services
- `monitoring/docker-compose.dev.yml` (update) -- dev-specific Fluent Bit config (stdout output for debugging)
- `monitoring/docker-compose.prod.yml` (update) -- prod-specific output config
- `monitoring/grafana/provisioning/datasources/` (update if needed) -- add Loki datasource for Grafana
- `monitoring/README.md` (update) -- document log aggregation setup
- `docs/COMPLIANCE.md` (update) -- mark centralized log aggregation gap as addressed

# Implementation Plan

1. Create `monitoring/fluent-bit/` directory with configuration files:
   - `fluent-bit.conf`: Configure Docker log input (read from `/var/lib/docker/containers/`), JSON parser for structured logs, and Loki output plugin
   - `parsers.conf`: JSON parser for structlog output, regex parser for standard container logs
2. Add Loki service to `monitoring/docker-compose.yml`:
   - Use `grafana/loki:latest` image
   - Expose port 3100
   - Mount persistent volume for log retention
   - Add health check
3. Add Fluent Bit service to `monitoring/docker-compose.yml`:
   - Use `fluent/fluent-bit:latest` image
   - Mount Docker socket or log directory as read-only
   - Mount `fluent-bit/` config directory
   - Depends on Loki service
   - Configure log driver for all existing services to use fluentd or json-file with labels
4. Add Loki as a Grafana datasource in provisioning config
5. Update `monitoring/docker-compose.dev.yml`:
   - Add stdout output alongside Loki for debugging visibility
6. Update `monitoring/docker-compose.prod.yml`:
   - Document environment variables for switching output to Datadog/CloudWatch/Elasticsearch
7. Update `monitoring/README.md` with log aggregation section:
   - How to view logs in Grafana via Loki
   - How to query logs with LogQL
   - How to configure production log backends
8. Update `docs/COMPLIANCE.md` to mark CC7.1 centralized log aggregation gap as addressed
