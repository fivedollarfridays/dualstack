# Objective

No external uptime monitoring exists -- a SOC2 CC7.1 gap. The backend exposes `/health/live` and `/health/ready` endpoints (used by Docker healthchecks), but no external system monitors these endpoints or alerts when they fail. Configure health check monitoring that detects downtime from outside the infrastructure and routes alerts via PagerDuty or OpsGenie. Document the setup for both managed uptime services (e.g., Pingdom, UptimeRobot, Better Uptime) and self-hosted alternatives (e.g., Prometheus blackbox_exporter).

# Files

- `monitoring/prometheus/alerts/uptime.yml` (new) -- Prometheus alert rules for health endpoint failures (for self-hosted monitoring)
- `monitoring/alertmanager/alertmanager.yml` (update) -- add PagerDuty receiver configuration (templated with envsubst)
- `docs/MONITORING.md` (new) -- comprehensive monitoring guide including uptime monitoring setup
- `docs/COMPLIANCE.md` (update) -- mark uptime monitoring gap as addressed
- `scripts/healthcheck.sh` (new, optional) -- standalone health check script for cron-based monitoring

# Implementation Plan

1. Create Prometheus alert rules for health endpoint monitoring:
   - `HealthLiveEndpointDown`: fires when `/health/live` returns non-200 for >1 minute (critical)
   - `HealthReadyEndpointDown`: fires when `/health/ready` returns non-200 for >2 minutes (critical)
   - `HealthEndpointSlow`: fires when health check response time >5s for >3 minutes (warning)
2. Update Alertmanager config to include PagerDuty receiver:
   - Add PagerDuty integration key placeholder (via `$PAGERDUTY_SERVICE_KEY` envsubst)
   - Route critical health alerts to PagerDuty
   - Route warning health alerts to Slack (existing config pattern)
   - Add inhibition rule: suppress `HealthReadyEndpointDown` when `HealthLiveEndpointDown` is firing
3. Create `docs/MONITORING.md` with sections:
   - Overview of monitoring architecture (Prometheus + Grafana + Alertmanager)
   - Health endpoints reference (`/health/live`, `/health/ready`, what each checks)
   - External uptime monitoring setup guide:
     - Option A: Managed service (Pingdom, UptimeRobot, Better Uptime) -- step-by-step
     - Option B: Self-hosted with Prometheus blackbox_exporter
   - PagerDuty integration guide:
     - Creating a PagerDuty service
     - Configuring the integration key
     - Alert routing and escalation
   - OpsGenie integration guide (alternative to PagerDuty)
   - Alert severity mapping (critical -> page, warning -> Slack, info -> log)
4. Create `scripts/healthcheck.sh`:
   - Curl both health endpoints
   - Exit non-zero on failure (usable in cron jobs or external monitoring)
   - Support `--url` flag for custom base URL
5. Update `docs/COMPLIANCE.md`:
   - Mark CC7.1 uptime monitoring gap as addressed
   - Note which monitoring option was chosen
