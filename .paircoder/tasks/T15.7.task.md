# Objective

DualStack is not a true monorepo -- there is no shared dependency management or coordinated task execution across the backend and frontend directories. Each stack is managed independently (`pip` for backend, `npm` for frontend), and there are no root-level scripts for cross-stack operations. Add workspace configuration using pnpm workspaces (or Turborepo) so that common operations (install, test, lint, build) can be run from the project root and dependencies are deduplicated where possible.

# Files

- `pnpm-workspace.yaml` (new) -- workspace definition including frontend (and optionally scripts)
- `package.json` (new, root-level) -- root package.json with workspace scripts for cross-stack operations
- `turbo.json` (new, optional) -- Turborepo pipeline config for parallel task execution and caching
- `frontend/package.json` (update, if needed) -- ensure `name` field is set for workspace resolution
- `README.md` (update) -- document monorepo workspace setup and root-level commands
- `.npmrc` (new, optional) -- pnpm configuration (e.g., `shamefully-hoist=true` if needed for Next.js)

# Implementation Plan

1. Install pnpm (document as a prerequisite if not already available):
   - pnpm is required for workspace support
   - Document install: `npm install -g pnpm` or `corepack enable`
2. Create `pnpm-workspace.yaml`:
   ```yaml
   packages:
     - 'frontend'
   ```
   - Note: backend is Python (pip-managed), so it is not a pnpm workspace package. Only JS/TS packages are included.
3. Create root `package.json`:
   - `name`: `dualstack`
   - `private`: true
   - `scripts`:
     - `dev`: Start both backend and frontend (using `concurrently` or separate commands)
     - `test`: Run backend tests (`cd backend && pytest`) and frontend tests (`pnpm --filter frontend test`)
     - `test:backend`: `cd backend && python -m pytest`
     - `test:frontend`: `pnpm --filter frontend test`
     - `lint`: Run linters for both stacks
     - `build`: Build frontend (`pnpm --filter frontend build`)
     - `install:all`: Install both backend (`cd backend && pip install -r requirements.txt`) and frontend (`pnpm install`) deps
   - `devDependencies`: `concurrently` (for parallel dev server startup)
4. Optionally create `turbo.json` for Turborepo:
   - Define pipeline for `build`, `test`, `lint` tasks
   - Configure caching for build outputs
   - Set task dependencies (e.g., build depends on lint)
5. Update `frontend/package.json`:
   - Ensure `name` field is set to `@dualstack/frontend` or `frontend` for workspace resolution
6. Create `.npmrc` if needed:
   - `shamefully-hoist=true` if Next.js requires hoisted dependencies
   - `engine-strict=true` to enforce Node.js version
7. Update `README.md`:
   - Add section on monorepo workspace setup
   - Document root-level commands (`pnpm install`, `pnpm test`, `pnpm dev`)
   - Note that backend Python deps are managed separately via pip
