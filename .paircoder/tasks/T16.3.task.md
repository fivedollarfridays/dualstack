# Objective

No admin interface exists. Create admin pages: user list (with role management), system health overview (metrics summary), and audit log viewer. Backend admin endpoints must be protected with the RBAC system from T16.2. Frontend admin pages must only be accessible to users with the admin role.

# Implementation Plan

1. Write tests for admin service -- user listing:
   - `list_users()` returns paginated user list with roles
   - `list_users(search="term")` filters by name or email
   - `assign_role(user_id, role)` updates the user's role
   - `assign_role()` raises error for invalid role values
2. Implement `admin/service.py` with user management methods
3. Write tests for admin routes:
   - `GET /api/admin/users` returns user list (admin only, 403 for members)
   - `PATCH /api/admin/users/{id}/role` updates user role (admin only)
   - `GET /api/admin/health` returns system health summary (admin only)
   - `GET /api/admin/audit` returns paginated audit log (admin only)
4. Implement `admin/routes.py` with role-protected endpoints using `require_role(Role.ADMIN)`
5. Write tests for frontend admin layout:
   - Renders children when user has admin role
   - Redirects to dashboard (or shows 403) when user is not admin
6. Implement frontend admin pages:
   - Admin layout with role check
   - User table with search and role assignment dropdown
   - Health overview displaying key metrics
   - Audit log with pagination controls
7. Write component tests for user-table, health-overview, and audit-log
