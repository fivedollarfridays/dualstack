# Objective

No user profile page exists. The settings page currently just links to Clerk. Add profile management: display name (editable), avatar (displayed from Clerk or uploaded), email (read-only), and account deletion (with confirmation dialog). The profile page should work in both Clerk-enabled and dev-bypass modes.

# Files

Backend source files:
- `backend/app/users/routes.py` -- add profile endpoints (`GET /api/users/me`, `PATCH /api/users/me`, `DELETE /api/users/me`)
- `backend/app/users/schemas.py` -- add `UserProfile`, `UserProfileUpdate` schemas
- `backend/app/users/service.py` -- add profile retrieval, update, and deletion logic

Frontend source files:
- `frontend/src/app/settings/profile/page.tsx` (new) -- profile management page
- `frontend/src/components/profile/profile-form.tsx` (new) -- editable profile form
- `frontend/src/components/profile/avatar-display.tsx` (new) -- avatar display component
- `frontend/src/components/profile/delete-account-dialog.tsx` (new) -- account deletion confirmation

Backend test files:
- `backend/tests/users/test_profile_routes.py` (new) -- tests for profile API endpoints
- `backend/tests/users/test_profile_service.py` (new) -- tests for profile business logic

Frontend test files:

# Implementation Plan

1. Write tests for profile service:
   - `get_profile(user_id)` returns user profile with name, email, avatar URL
   - `update_profile(user_id, data)` updates display name
   - `delete_account(user_id)` soft-deletes or fully removes user and associated data
   - `delete_account()` raises error if user not found
2. Implement `users/service.py` profile methods
3. Write tests for profile routes:
   - `GET /api/users/me` returns authenticated user's profile
   - `PATCH /api/users/me` updates display name, returns updated profile
   - `DELETE /api/users/me` deletes account (requires confirmation token/header)
   - All endpoints return 401 for unauthenticated requests
4. Implement profile endpoints in `users/routes.py`
5. Write tests for frontend components:
   - `ProfileForm` renders current name in input, submits update on save
   - `AvatarDisplay` shows user avatar image or initials fallback
   - `DeleteAccountDialog` shows confirmation with typed confirmation, calls delete on confirm
6. Implement frontend components and profile page
