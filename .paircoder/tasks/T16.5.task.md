# Objective

The items list endpoint has pagination but no search, sort, or filter capabilities. Add query parameters for text search (by name/description), sort by field (created_at, name, updated_at), sort direction (asc/desc), and filter by criteria (e.g., status, date range). Update the frontend to expose search bar, sort dropdown, and filter controls.

# Files

Backend source files:
- `backend/app/items/routes.py` -- add `search`, `sort_by`, `sort_dir`, `filter_*` query parameters to list endpoint
- `backend/app/items/service.py` -- implement search, sort, and filter query building logic
- `backend/app/items/schemas.py` -- add `ItemListParams` schema for query validation

Frontend source files:
- `frontend/src/hooks/useItems.ts` (or equivalent) -- pass search/sort/filter params to API calls
- `frontend/src/components/items/search-bar.tsx` (new) -- search input with debounce
- `frontend/src/components/items/sort-controls.tsx` (new) -- sort field and direction selector
- `frontend/src/components/items/filter-controls.tsx` (new) -- filter UI controls

Backend test files:
- `backend/tests/items/test_items_search.py` (new) -- tests for search functionality
- `backend/tests/items/test_items_sort_filter.py` (new) -- tests for sort and filter

Frontend test files:

# Implementation Plan

1. Write tests for search functionality:
   - `GET /api/items?search=foo` returns items where name or description contains "foo"
   - Search is case-insensitive
   - Empty search string returns all items (no filter applied)
   - Search with no matches returns empty list with `total: 0`
2. Implement search query building in `items/service.py`
3. Write tests for sort functionality:
   - `GET /api/items?sort_by=name&sort_dir=asc` returns items sorted by name ascending
   - `GET /api/items?sort_by=created_at&sort_dir=desc` returns newest first
   - Invalid `sort_by` field returns 422 validation error
   - Default sort is `created_at` descending
4. Implement sort query building
5. Write tests for filter functionality:
   - Filter parameters documented in OpenAPI schema
   - Filters combine with search and sort (all query params work together)
6. Implement filter logic and update routes with validated query params
7. Write frontend component tests:
   - SearchBar calls onChange with debounced value
   - SortControls renders field options and direction toggle
8. Implement frontend search/sort/filter UI components
9. Wire up hook to pass all query params to the API
