# Objective

No file upload capability exists. Add presigned URL generation for S3-compatible storage (Cloudflare R2 recommended for cost efficiency, with AWS S3 as an alternative). The backend generates presigned upload URLs and tracks file metadata. The frontend handles direct-to-storage upload with progress indication. The storage backend must be configurable via env vars.

# Implementation Plan

1. Write tests for storage service:
   - `generate_upload_url(key, content_type, expires_in)` returns a presigned URL string
   - `generate_download_url(key, expires_in)` returns a presigned download URL
   - Raises `StorageConfigError` when credentials are not configured
   - Works with both S3 and R2 endpoint configurations
2. Implement `storage.py` using `boto3` with configurable endpoint URL for R2 compatibility
3. Write tests for file metadata model:
   - File model stores key, filename, size, content_type, user_id, created_at
   - Unique constraint on storage key
4. Implement file model and migration
5. Write tests for file routes:
   - `POST /api/files/upload-url` returns presigned URL and file ID (authenticated)
   - `GET /api/files` returns user's files with pagination
   - `GET /api/files/{id}/download-url` returns presigned download URL
   - `DELETE /api/files/{id}` removes file metadata (and optionally the object)
6. Implement file routes and service
7. Write frontend component tests:
   - `FileUpload` renders drop zone, calls onUpload with selected file
   - `UploadProgress` shows progress percentage and filename
8. Implement frontend upload components and hook
