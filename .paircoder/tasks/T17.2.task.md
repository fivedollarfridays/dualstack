# Objective

No real-time features exist. Add WebSocket support using FastAPI's native WebSocket capabilities for live updates (e.g., item changes, notifications). The backend broadcasts events when items are created, updated, or deleted. The frontend maintains a persistent WebSocket connection with automatic reconnection. Authentication is enforced on WebSocket connections.

# Files

Backend source files:
- `backend/app/core/websocket.py` (new) -- WebSocket connection manager (track connections, broadcast)
- `backend/app/core/events.py` (new) -- event definitions and event bus for publishing changes
- `backend/app/core/ws_auth.py` (new) -- WebSocket authentication (verify token from query param or first message)
- `backend/app/items/routes.py` -- add event publishing on item create/update/delete

Frontend source files:
- `frontend/src/hooks/useWebSocket.ts` (new) -- hook for WebSocket connection with reconnection logic
- `frontend/src/hooks/useRealtimeItems.ts` (new) -- hook that updates item list on real-time events

Backend test files:
- `backend/tests/core/test_websocket_manager.py` (new) -- tests for connection manager
- `backend/tests/core/test_events.py` (new) -- tests for event bus
- `backend/tests/core/test_ws_auth.py` (new) -- tests for WebSocket authentication

Frontend test files:
- `frontend/src/hooks/__tests__/useWebSocket.test.ts` (new)
- `frontend/src/hooks/__tests__/useRealtimeItems.test.ts` (new)

# Implementation Plan

1. Write tests for the connection manager:
   - `ConnectionManager.connect(websocket, user_id)` adds connection to active set
   - `ConnectionManager.disconnect(websocket)` removes connection
   - `ConnectionManager.broadcast(event)` sends to all connected clients
   - `ConnectionManager.send_to_user(user_id, event)` sends to a specific user's connections
2. Implement `websocket.py` with `ConnectionManager` class
3. Write tests for the event bus:
   - `EventBus.publish("item.created", payload)` notifies all subscribers
   - `EventBus.subscribe("item.*", handler)` registers a handler for matching events
   - Event types: `item.created`, `item.updated`, `item.deleted`
4. Implement `events.py` with `EventBus` and event type definitions
5. Write tests for WebSocket authentication:
   - Valid token in query param allows connection
   - Missing or invalid token closes connection with 4001 code
   - Expired token closes connection with 4001 code
6. Implement `ws_auth.py` to verify auth tokens on WebSocket handshake
7. Wire event publishing into item routes (create/update/delete)
8. Write frontend hook tests:
   - `useWebSocket` connects to WS endpoint with auth token
   - `useWebSocket` reconnects after disconnection with exponential backoff
   - `useRealtimeItems` updates item list when `item.created` event received
9. Implement frontend hooks

# Acceptance Criteria
