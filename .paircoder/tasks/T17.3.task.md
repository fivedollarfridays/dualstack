# Objective

No onboarding experience exists for new users. Add a guided setup wizard that appears after signup. The wizard walks users through: welcome step (intro to the app), configure preferences (theme, notification settings), create first item (guided item creation), and completion (success confirmation). The wizard can be skipped at any step. Completion is tracked so it does not appear again for returning users.

# Implementation Plan

1. Write tests for `useOnboarding` hook:
   - Returns current step index (starting at 0)
   - `nextStep()` advances to the next step
   - `prevStep()` goes back to the previous step
   - `skip()` marks onboarding as complete and redirects to dashboard
   - `complete()` marks onboarding as complete (persists to localStorage or user metadata)
   - Returns `isComplete: true` when onboarding was previously completed
2. Implement `useOnboarding` hook
3. Write tests for `WizardContainer`:
   - Renders the current step component based on step index
   - Shows progress indicator (e.g., "Step 2 of 4")
   - Shows "Skip" button on every step
   - Shows "Next" and "Back" buttons appropriately
4. Implement `WizardContainer`
5. Write tests for each step component:
   - `WelcomeStep` renders welcome heading and app description
   - `PreferencesStep` renders preference controls (theme toggle, notification opt-in)
   - `FirstItemStep` renders a simplified item creation form
   - `CompletionStep` renders success message and "Go to Dashboard" button
6. Implement step components
7. Wire up the onboarding page to redirect users who have already completed onboarding
