# Objective

The landing page is a single static page with no pricing information or blog. Add a pricing page with plan comparison (free/pro/enterprise tiers) and a blog section powered by MDX content. The pricing page should be data-driven so plan details can be updated without code changes. The blog should support MDX files with frontmatter for title, date, author, and description.

# Implementation Plan

1. Write tests for pricing data and components:
   - `pricing-plans.ts` exports an array of plan objects with name, price, features, and CTA
   - `PricingCard` renders plan name, price, feature list, and CTA button
   - `PricingTable` renders all plans side by side with feature comparison
   - Active/recommended plan is visually highlighted
2. Implement pricing data, PricingCard, and PricingTable components
3. Write tests for blog utilities:
   - `getBlogPosts()` returns array of posts sorted by date descending
   - Each post has slug, title, date, author, description, and content
   - `getBlogPost(slug)` returns a single post or null for unknown slug
4. Implement `blog.ts` with MDX file loading and frontmatter parsing
5. Write tests for blog components:
   - `BlogCard` renders post title, date, description, and link to full post
6. Implement blog index page and individual post page
7. Add SEO metadata (title, description, Open Graph) to both pricing and blog pages
8. Create 2 sample MDX blog posts with proper frontmatter
