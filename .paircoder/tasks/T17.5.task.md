# Objective

No SEO basics are implemented. Add dynamic `sitemap.xml` generation that includes all public routes (landing, pricing, blog posts), a `robots.txt` configuration that allows search engine crawling of public pages while blocking authenticated routes, and enhanced metadata (Open Graph, Twitter cards) on all public-facing pages.

# Files

Frontend source files:
- `frontend/src/app/sitemap.ts` (new) -- Next.js dynamic sitemap generation
- `frontend/src/app/robots.ts` (new) -- Next.js robots.txt generation
- `frontend/src/app/layout.tsx` -- enhance root metadata with site-wide defaults
- `frontend/src/lib/seo.ts` (new) -- SEO helper utilities (generate metadata objects, canonical URLs)

Frontend test files:
- `frontend/src/lib/__tests__/seo.test.ts` (new) -- tests for SEO helper utilities
- `frontend/src/app/__tests__/sitemap.test.ts` (new) -- tests for sitemap generation
- `frontend/src/app/__tests__/robots.test.ts` (new) -- tests for robots.txt generation

# Implementation Plan

1. Write tests for SEO helper utilities:
   - `generateMetadata(page)` returns a metadata object with title, description, Open Graph, and Twitter card fields
   - `getCanonicalUrl(path)` returns the full canonical URL using the configured base URL
   - Metadata includes `og:title`, `og:description`, `og:image`, `og:url`
   - Twitter card metadata includes `twitter:card`, `twitter:title`, `twitter:description`
2. Implement `seo.ts` with metadata generation helpers
3. Write tests for sitemap generation:
   - `sitemap()` returns an array of URL entries
   - Includes static routes: `/`, `/pricing`, `/blog`
   - Includes dynamic routes: `/blog/[slug]` for each blog post
   - Each entry has `url`, `lastModified`, `changeFrequency`, `priority`
   - Does NOT include authenticated routes (`/items`, `/settings`, `/admin`)
4. Implement `sitemap.ts` using Next.js `MetadataRoute.Sitemap` return type
5. Write tests for robots.txt:
   - Allows crawling of `/`, `/pricing`, `/blog`
   - Disallows crawling of `/items`, `/settings`, `/admin`, `/api`
   - Includes sitemap URL reference
6. Implement `robots.ts` using Next.js `MetadataRoute.Robots` return type
7. Update root `layout.tsx` metadata with site-wide defaults (title template, description, icons)

# Acceptance Criteria

- [ ] `sitemap.xml` is auto-generated at `/sitemap.xml` with all public routes
