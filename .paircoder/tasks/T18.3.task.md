# Objective

Fix broken access control: WebSocket broadcasts all item events to all connected users. Events should only be sent to the item's owning user.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/core/ws_routes.py` | Change `_broadcast_event` to use `send_to_user` |
| `backend/app/items/service.py` | Ensure all `event_bus.publish` calls include `user_id` |
| `backend/tests/core/test_websocket_manager.py` | Add user isolation tests |

# Implementation Plan

1. Write failing test: User A connected, User B creates item → User A should NOT receive the event
2. Check all `event_bus.publish()` calls in `items/service.py` — add `user_id` to payload if missing
3. Modify `_broadcast_event` in `ws_routes.py` to extract `user_id` and call `manager.send_to_user()`
4. Run WebSocket manager tests + item tests

# Acceptance Criteria

- [x] Item events routed only to the owning user
- [x] `manager.broadcast()` no longer called for item events
- [x] All `event_bus.publish` calls include `user_id` in payload
- [x] Test: User A does not receive User B's item events
- [x] Existing WebSocket tests pass
