# Objective

Replace 3 raw `HTTPException` raises with custom `AppError` subclasses for consistent API error responses. Also fix billing route type annotations, audit persistence, and FileResponse Pydantic config.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/billing/routes.py` | Replace `HTTPException(404)` with `NotFoundError`, add return types, switch to `persist_audit_event` |
| `backend/app/core/entitlements.py` | Replace `HTTPException(403)` with `AuthorizationError` |
| `backend/app/core/metrics_routes.py` | Replace `HTTPException(403)` with `AuthorizationError` |
| `backend/app/files/schemas.py` | Change `class Config` to `model_config` dict |

# Implementation Plan

1. Write tests expecting `{"error": {"code", "message"}}` from billing 404, entitlements 403, metrics 403
2. Replace `HTTPException` with `NotFoundError`/`AuthorizationError`
3. Add return type annotations to billing route handlers
4. Switch billing `log_audit_event` → `persist_audit_event`
5. Fix `FileResponse` Pydantic v1 `class Config` → v2 `model_config`
6. Run all tests
