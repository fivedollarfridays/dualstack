# Objective

Close two input validation gaps: avatar_url accepts `javascript:`/`data:` schemes (XSS vector), and file upload filenames aren't sanitized against path traversal.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/users/schemas.py` | Add `field_validator` for `avatar_url` requiring `https://` |
| `backend/app/files/service.py` | Sanitize `filename` with `PurePosixPath.name` |
| `backend/app/core/rate_limit.py` | Add docstring noting `--forwarded-allow-ips` requirement |

# Implementation Plan

1. Write failing tests for avatar URL validation and filename sanitization
2. Add `field_validator("avatar_url")` in `users/schemas.py` requiring `https://` scheme
3. Apply `PurePosixPath(filename).name` in `files/service.py` before constructing `storage_key`
4. Add production docstring to `get_client_ip` in `rate_limit.py`
5. Run all tests

# Acceptance Criteria

- [x] `avatar_url` rejects `javascript:`, `data:`, `http://` schemes
- [x] `avatar_url` accepts valid `https://` URLs and None/empty
- [x] Filename `../../etc/passwd` produces key with just `passwd`
- [x] Tests for all edge cases
- [x] All existing tests pass
