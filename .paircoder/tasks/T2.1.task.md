# Objective

Fix two critical security issues: (1) the portal endpoint accepts any customer_id without ownership verification (IDOR), and (2) missing clerk_jwks_url in production silently falls back to dev-mode auth bypass.

# Implementation Plan

## Fix 1: Portal IDOR

Since this is a starter kit with no user-customer mapping table, the pragmatic approach is to look up the Stripe customer server-side instead of trusting client input.

1. **Remove `customer_id` from `PortalRequest` schema** in `backend/app/billing/schemas.py`
2. **Add `get_customer_for_user()` to `backend/app/billing/service.py`** — uses `stripe.Customer.list(metadata={"user_id": user_id}, limit=1)` via `asyncio.to_thread()`
3. **Update `create_portal_session()`** to accept `user_id` instead of `customer_id`, call `get_customer_for_user()` internally
4. **Update `backend/app/billing/routes.py`** — pass `user_id` (from auth dependency) to service
5. **Update frontend** — remove `customerId` param from `openPortal()`, remove broken empty-string call from billing page
6. **Update all tests** — remove customer_id from schemas/routes tests, add customer lookup tests

## Fix 2: clerk_jwks_url startup guard

Add to `lifespan()` in `backend/app/main.py`, alongside the existing webhook secret guard:

```python
if settings.environment != "development" and not settings.clerk_jwks_url:
    raise RuntimeError(
        "CLERK_JWKS_URL must be set in non-development environments"
    )
```
