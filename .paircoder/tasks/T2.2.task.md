# Objective

Harden URL validation with explicit scheme enforcement and generic error messages. Fix timing-safe comparison for metrics API key. Expand sensitive params list. Add price_id format validation. Validate X-Correlation-ID format.

# Implementation Plan

## 1. URL validation hardening (`backend/app/core/url_validation.py`)

- Add explicit scheme check: `if parsed.scheme not in ("http", "https"): raise ValueError(...)`
- Add netloc check: `if not parsed.netloc: raise ValueError(...)`
- Change error message to generic: `"URL origin is not allowed"` (no leaking allowed list)

## 2. Metrics timing-safe comparison (`backend/app/core/metrics_routes.py`)

- Import `hmac`
- Replace `x_metrics_key != settings.metrics_api_key` with `not hmac.compare_digest(x_metrics_key or "", settings.metrics_api_key)`

## 3. Expand SENSITIVE_PARAMS (`backend/app/core/middleware.py`)

- Add to set: `"auth"`, `"authorization"`, `"apikey"`, `"code"`, `"state"`, `"client_secret"`, `"private_key"`, `"credential"`

## 4. X-Correlation-ID validation (`backend/app/core/middleware.py`)

- Add UUID regex: `_UUID_RE = re.compile(r'^[0-9a-f]{8}-...$', re.IGNORECASE)`
- Validate client-supplied correlation ID before using it
- Generate new UUID if client value doesn't match pattern

## 5. price_id validation (`backend/app/billing/schemas.py`)

- Add `field_validator("price_id")` checking `^price_[a-zA-Z0-9]+$`
