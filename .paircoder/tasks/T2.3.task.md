# Objective

Add proper error handling to all async event handlers using `mutateAsync` and fix the billing page portal button. Also improve `handleResponse` type safety.

# Implementation Plan

## 1. Items edit page (`frontend/src/app/(dashboard)/items/[id]/page.tsx`)

Wrap `handleSubmit` and `handleDelete` in try/catch:
```typescript
async function handleDelete() {
  if (window.confirm('...')) {
    try {
      await deleteItemMutation.mutateAsync(id);
      router.push('/items');
    } catch {
      // React Query tracks error via mutation.error
    }
  }
}
```

Add error UI using `updateItem.error` / `deleteItemMutation.error`.

## 2. Items list page (`frontend/src/app/(dashboard)/items/page.tsx`)

Same try/catch for `handleDelete`.

## 3. Billing page (`frontend/src/app/(dashboard)/billing/page.tsx`)

- After T2.1 lands: `openPortal(token, returnUrl)` — no customer_id
- Add try/catch to both `handleSubscribe` and `handleManageSubscription`
- Add error state display

## 4. handleResponse type overloads (`frontend/src/lib/api/shared.ts`)

```typescript
export async function handleResponse(response: Response): Promise<void>;
export async function handleResponse<T>(response: Response): Promise<T>;
export async function handleResponse<T>(response: Response): Promise<T | void> {
  if (!response.ok) throw new Error(`API error: ${response.status} ${response.statusText}`);
  if (response.status === 204) return;
  return response.json();
}
```

Update `deleteItem` in items.ts to call without generic type parameter.
