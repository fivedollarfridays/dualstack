# Objective

Transfer the 11 generic multi-tenant modules from Qluo to TenantForge: auth, billing, rbac, organizations, sso, admin, compliance, search, storage, users, public-api. Transfer K8s configs and wrangler deployment config. Strip all Qluo-specific imports and references. Every module must compile independently with zero references to CRM concepts.

# Files

Source files (from Qluo):
- `backend/modules/auth/` -- authentication module
- `backend/modules/billing/` -- billing/subscription module
- `backend/modules/rbac/` -- role-based access control
- `backend/modules/organizations/` -- organization/tenant management
- `backend/modules/sso/` -- single sign-on
- `backend/modules/admin/` -- admin panel
- `backend/modules/compliance/` -- compliance/audit
- `backend/modules/search/` -- search functionality
- `backend/modules/storage/` -- file storage
- `backend/modules/users/` -- user management
- `backend/modules/public-api/` -- public API layer
- K8s configs and wrangler deployment config

# Implementation Plan

1. Copy each of the 11 modules from Qluo to TenantForge repo
2. Strip all Qluo-specific imports and references (CRM concepts: leads, contacts, campaigns, deals, pipeline, etc.)
3. Transfer K8s configs and wrangler deployment config
4. Verify each module compiles independently
5. Grep for any remaining CRM references and remove them
