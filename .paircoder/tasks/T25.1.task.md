# Objective

Enforce JWT audience validation to prevent cross-app token replay attacks.

# Implementation Plan

1. Add `CLERK_AUDIENCE` startup check in `main.py` — RuntimeError when absent in production (if CLERK_JWKS_URL is set)
2. Add startup warning when `CLERK_AUDIENCE` is empty in non-production
3. In `ws_auth.py`, remove the `verify_aud: False` bypass — always pass audience to decode when configured
4. Update `.env.example` with `CLERK_AUDIENCE` variable

# Acceptance Criteria

- [ ] Production startup fails with RuntimeError if `CLERK_JWKS_URL` is set but `CLERK_AUDIENCE` is not
- [ ] Non-production emits warning when `CLERK_AUDIENCE` is not set
- [ ] WebSocket JWT decode verifies audience when `CLERK_AUDIENCE` is configured
- [ ] WebSocket JWT decode still works in dev mode without audience (backward compat)
- [ ] `.env.example` includes `CLERK_AUDIENCE`
- [ ] Tests cover: startup failure, warning, audience pass, audience fail, dev mode
