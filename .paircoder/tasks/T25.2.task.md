# Objective

Replace wildcard `wss: ws:` in CSP connect-src with specific backend origins.

# Implementation Plan

1. In `csp.ts`, read `NEXT_PUBLIC_API_URL` and `NEXT_PUBLIC_WS_URL` env vars
2. Extract origin (protocol + host) from each URL
3. Build connect-src with `'self'`, Clerk domains, Stripe, and the extracted backend origins
4. Drop bare `ws:` entirely — production must use `wss://`
5. Add fallback for dev (localhost origins)

# Acceptance Criteria

- [ ] CSP connect-src contains specific backend origins, not wildcard `wss:` or `ws:`
- [ ] Production CSP only allows `wss://` backend origins
- [ ] Dev CSP allows `ws://localhost:*` for local development
- [ ] Clerk and Stripe domains still present in connect-src
- [ ] Frontend build succeeds
- [ ] Tests verify no wildcard ws:/wss: in CSP output
