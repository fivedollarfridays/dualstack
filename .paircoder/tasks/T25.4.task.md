# Objective

Move WebSocket JWT from URL query parameter to first-message pattern to prevent token leakage in logs/proxies.

# Implementation Plan

**Backend:**
1. In `ws_routes.py`, accept the WebSocket first (no auth), then wait for first message
2. Parse first message as JSON `{"type":"auth","token":"<jwt>"}`
3. Call existing `authenticate_ws` logic on the extracted token
4. Send `{"type":"auth_ok"}` on success, close with 4001 on failure
5. Add 5-second timeout — close if no auth message received
6. Enter normal receive loop only after successful auth

**Frontend:**
1. In `use-websocket.ts`, connect without token in URL
2. On `onopen`, send `JSON.stringify({type:"auth",token})` as first message
3. Wait for `{"type":"auth_ok"}` before setting status to `connected`
4. Handle auth failure (close event with 4001)

# Acceptance Criteria

- [ ] WebSocket URL contains no token query parameter
- [ ] First message from client is `{"type":"auth","token":"..."}`
- [ ] Server responds with `{"type":"auth_ok"}` on success
- [ ] Server closes with 4001 if auth fails or times out (5s)
- [ ] Existing realtime-items integration works end-to-end
- [ ] Tests cover: successful auth, timeout, invalid token, missing auth message
