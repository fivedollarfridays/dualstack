# Objective

Remove Turso auth token from Alembic connection URL string to prevent log leaks.

# Implementation Plan

1. Check if `sqlalchemy-libsql` supports `connect_args={"auth_token": token}`
2. If yes: use `connect_args` instead of embedding `?authToken=` in URL
3. If no: add a startup assertion that `echo=False` when using remote Turso, and add URL scrubbing for `authToken=` in log output
4. Add a production guard: RuntimeError if `LOG_LEVEL=DEBUG` with remote Turso URL

# Acceptance Criteria

- [ ] Auth token does not appear in the Alembic connection URL string
- [ ] Alembic migrations still work with remote Turso
- [ ] Local file-based Turso URLs are unaffected
- [ ] Tests verify token is not in URL when using connect_args pattern
- [ ] All existing migration and database tests pass
