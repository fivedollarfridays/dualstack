# Objective

Add message size limit to WebSocket post-auth receive loop to prevent memory exhaustion.

# Implementation Plan

1. Add `MAX_WS_MESSAGE_SIZE = 64 * 1024` constant (64 KB — messages are just pings)
2. In `ws_routes.py` receive loop, check `len(data) > MAX_WS_MESSAGE_SIZE` and close with 4009 if exceeded
3. Add tests for oversized message rejection

# Acceptance Criteria

- [ ] Messages exceeding 64 KB cause connection close with code 4009
- [ ] Normal ping/pong messages work unchanged
- [ ] Tests cover: oversized rejection, normal message pass
- [ ] All existing WS tests pass

# Files to Modify

- `backend/app/core/ws_routes.py`
- `backend/tests/core/test_ws_routes.py`
