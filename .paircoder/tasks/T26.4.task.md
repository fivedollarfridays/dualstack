# Objective

Add IP-based rate limiting on WebSocket upgrade requests to prevent connection exhaustion.

# Implementation Plan

1. Add a simple in-memory IP rate tracker (dict of IP -> recent connection timestamps)
2. Before `websocket.accept()` in `ws_routes.py`, check if IP has exceeded 30 connections/minute
3. If exceeded, close with 4029 (too many requests) without accepting
4. Clean up stale entries periodically

# Acceptance Criteria

- [ ] IP exceeding 30 WS connections/minute gets rejected with 4029
- [ ] Normal connection rate works unchanged
- [ ] Rate tracker cleans up stale entries
- [ ] Tests cover: rate limit hit, normal rate pass, cleanup
- [ ] All existing WS tests pass

- `backend/app/core/ws_routes.py`
- `backend/tests/core/test_ws_routes.py`
