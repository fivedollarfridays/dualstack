# Objective

Remove ingest-time display_name sanitization; enforce context-aware escaping at all output sinks.

# Implementation Plan

1. Remove the `<>` stripping validator from `UserProfileUpdate.display_name` in `schemas.py`
2. Store raw Unicode — trust Pydantic's `max_length=255` for size bounds
3. Verify React JSX auto-escapes display_name in all frontend components
4. Add `html.escape()` to email templates if display_name is used there
5. Add backend test: raw `<script>` stored correctly, returned correctly
6. Add frontend test: display_name renders as text (not HTML)

# Acceptance Criteria

- [ ] `display_name` stored as raw Unicode (no stripping)
- [ ] Frontend renders display_name safely via JSX auto-escaping
- [ ] Email templates (if any) escape display_name
- [ ] Admin dashboard renders display_name safely
- [ ] Tests cover: raw storage, safe rendering, email escaping
- [ ] All existing tests pass
