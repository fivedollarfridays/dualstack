# Objective

The `alembic/versions/` directory is empty. The quickstart tells users to run `alembic upgrade head` but there's no migration to run. Generate the initial migration for the Item model so the quickstart actually works.

# Implementation Plan

## 1. Write the initial migration manually

Create `backend/alembic/versions/001_initial_items.py` with:
- `items` table matching the Item model (id, user_id, title, description, status, created_at, updated_at)
- Proper `upgrade()` and `downgrade()` functions
- Index on `user_id`

Writing manually (not autogenerate) ensures the migration is clean and doesn't depend on local DB state.

## 2. Verify migration runs

```bash
cd backend && alembic upgrade head
cd backend && alembic downgrade base
cd backend && alembic upgrade head
```

### Files to create

- `backend/alembic/versions/001_initial_items.py`
