# Objective

The billing portal flow is half-working: the backend has a portal endpoint with ownership verification, but the frontend has `customerId = ''` hardcoded — so it never works. The user explicitly stated "half-working features are worse than documented gaps."

**Decision**: Simplify the portal to a documented placeholder. Keep checkout + webhooks (they work). Remove the pretense that the portal works today.

# Implementation Plan

## 1. Backend: Simplify portal service (`backend/app/billing/service.py`)

Remove the ownership verification complexity. Simplify to just create a portal session — no ownership check. Remove `user_id` parameter. Add a docstring explaining the missing user→customer mapping.

```python
async def create_portal_session(customer_id: str, return_url: str) -> str:
    """Create a Stripe Billing Portal session.

    NOTE: To use this in production, you need a user→customer mapping.
    Store the Stripe customer ID from the checkout.session.completed webhook
    in your database, then look it up here instead of accepting it as input.
    """
    session = await asyncio.to_thread(
        stripe.billing_portal.Session.create,
        customer=customer_id,
        return_url=return_url,
    )
    return session.url
```

Remove unused `AuthorizationError`, `NotFoundError` imports.

## 2. Backend: Simplify portal route (`backend/app/billing/routes.py`)

Update to pass `data.customer_id` and `data.return_url` directly (no `user_id`).

## 3. Frontend: Remove non-working portal button (`frontend/src/app/(dashboard)/billing/page.tsx`)

- Remove `handleManageSubscription` function
- Remove the `openPortal` import
- Remove the "Manage Subscription" button
- Add a TODO comment: `// TODO: Add "Manage Subscription" button once you have a user→customer mapping`

## 4. Frontend: Simplify billing API (`frontend/src/lib/api/billing.ts`)

Remove `openPortal` function. Add a TODO comment.

## 5. Update tests

- Backend `test_service.py`: Remove 3 ownership tests, add 1 simple portal test
