# Objective

Create `src/lib/auth-config.ts` with `isClerkEnabled()`, `DEV_USER_ID='dev-user-001'`, `DEV_TOKEN='dev-token'` constants. Create `src/contexts/auth-context.tsx` with `AuthContext`, `AuthContextValue` interface, and `useAppAuth()` hook. TDD -- tests first.

# Files

- `src/lib/auth-config.ts` -- configuration helper and dev constants
- `src/lib/auth-config.test.ts` -- tests for auth-config
- `src/contexts/auth-context.tsx` -- React context and hook
- `src/contexts/auth-context.test.tsx` -- tests for auth-context

# Implementation Plan

1. Write tests for `isClerkEnabled()`:
   - Returns `false` when `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` is undefined
   - Returns `false` when key is empty string
   - Returns `false` when key is an invalid string (e.g. `"not-a-key"`)
   - Returns `true` when key starts with `pk_test_`
   - Returns `true` when key starts with `pk_live_`
2. Implement `auth-config.ts` with `isClerkEnabled()`, `DEV_USER_ID`, `DEV_TOKEN`
3. Write tests for `AuthContext` and `useAppAuth()`:
   - `useAppAuth()` throws an error when used outside `AuthContext.Provider`
   - `useAppAuth()` returns the context value when inside a provider
4. Implement `auth-context.tsx` with `AuthContextValue` interface, `AuthContext`, and `useAppAuth()` hook

# Acceptance Criteria

- [ ] `isClerkEnabled()` returns `false` when `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` is missing, empty, or invalid
- [ ] `isClerkEnabled()` returns `true` for valid `pk_test_` or `pk_live_` prefixed keys
- [ ] `DEV_USER_ID` equals `'dev-user-001'` and `DEV_TOKEN` equals `'dev-token'`
- [ ] `useAppAuth()` throws a descriptive error when used outside `AuthContext.Provider`
- [ ] `useAppAuth()` returns `AuthContextValue` when used inside a provider
- [ ] All tests pass with `npm test`
