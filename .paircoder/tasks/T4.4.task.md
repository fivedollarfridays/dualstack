# Objective

Update `src/middleware.ts` to check `isClerkEnabled()` -- if false, return `NextResponse.next()` (passthrough). Dynamic-import `@clerk/nextjs/server` only when enabled. Update `src/lib/auth.ts` to check `isClerkEnabled()` -- if false, return dev user stubs. Update `src/components/providers.tsx` to remove `isValidClerkKey`/`ClerkNotConfiguredError` and branch between `ClerkProvider`+`ClerkAuthBridge` and `DevAuthProvider`.

# Files

- `src/middleware.ts` -- conditional Clerk middleware
- `src/lib/auth.ts` -- server-side auth helper
- `src/lib/auth.test.ts` -- tests for auth.ts
- `src/components/providers.tsx` -- provider tree with conditional branching
- `src/components/providers.test.tsx` -- tests for providers.tsx

# Implementation Plan

1. Update `auth.ts`:
   - Import `isClerkEnabled` from `auth-config`
   - When disabled: return `{ userId: DEV_USER_ID }` without calling Clerk
   - When enabled: call Clerk `auth()` as before
2. Write/update `auth.test.ts`:
   - Test dev mode: returns `dev-user-001` without Clerk
   - Test Clerk mode: calls Clerk auth and returns result
3. Run `npm test -- --testPathPattern='lib/auth'`
4. Update `middleware.ts`:
   - Import `isClerkEnabled` from `auth-config`
   - When disabled: export middleware that returns `NextResponse.next()`
   - When enabled: dynamic-import `@clerk/nextjs/server` and use `clerkMiddleware`
5. Update `providers.tsx`:
   - Remove `isValidClerkKey` function and `ClerkNotConfiguredError` component
   - Import `isClerkEnabled` from `auth-config`
   - Import `ClerkAuthBridge` and `DevAuthProvider`
   - When enabled: wrap children in `ClerkProvider` > `ClerkAuthBridge`
   - When disabled: wrap children in `DevAuthProvider`
6. Update `providers.test.tsx`:
   - Test dev mode: renders `DevAuthProvider` (no Clerk)
   - Test Clerk mode: renders `ClerkProvider` + `ClerkAuthBridge`
   - Remove tests for `ClerkNotConfiguredError`
7. Run full test suite
