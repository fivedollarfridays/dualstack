# Objective

Replace CRM entities (leads, contacts) with a generic "Items" entity matching the EdgeStack/DualStack pattern. Build full CRUD: list, create, read, update, delete. The Items entity must demonstrate the multi-tenant pattern: items scoped to organization with RBAC enforcement.

# Files

Source files:
- `backend/modules/items/` (new) -- Items module directory
- `backend/modules/items/model.ts` (new) -- Items data model
- `backend/modules/items/service.ts` (new) -- Items business logic
- `backend/modules/items/routes.ts` (new) -- Items API routes
- `backend/modules/items/types.ts` (new) -- Items type definitions

Test files:
- `backend/tests/modules/items/` (new) -- Items test directory

# Implementation Plan

1. Write tests for Items CRUD operations (list, create, read, update, delete)
2. Write tests for org-scoped access control (items belong to organizations, RBAC enforced)
3. Implement Items model with organization scope
4. Implement Items service with CRUD operations
5. Implement Items API routes
6. Verify all tests pass with org-scoped access control

# Acceptance Criteria

- [ ] Items entity exists with full CRUD: list, create, read, update, delete
- [ ] Items are scoped to organizations (multi-tenant isolation)
- [ ] RBAC enforcement on all Items operations
- [ ] Items demonstrates the multi-tenant pattern matching EdgeStack/DualStack
- [ ] All Items CRUD tests pass

# Verification

- Items CRUD tests pass
- Items are org-scoped (cannot access items from another organization)
- RBAC is enforced on all operations
