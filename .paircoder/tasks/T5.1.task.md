# Objective

The monitoring stack was extracted from a "Companion AI" application and still contains 35+ references to that project, plus alerts/dashboards for a "Video Generation" feature that doesn't exist in DualStack. This must be cleaned before shipping to buyers. Covers issues C2, C3, H5, H6, M12, M17.

# Implementation Plan

## 1. Replace "Companion AI" branding with "DualStack" (13 files)

Search and replace across all monitoring files:
- `companion-prometheus` -> `dualstack-prometheus`
- `companion-alertmanager` -> `dualstack-alertmanager`
- `companion-grafana` -> `dualstack-grafana`
- `companion-api` -> `dualstack-api` (Prometheus job names)
- `companion-ai` -> `dualstack` (cluster labels, dashboard tags)
- `companion-backend-network` -> `dualstack-backend-network`
- `https://github.com/companion-ai/runbooks/` -> remove or replace with generic placeholder

**Files:** docker-compose.yml, docker-compose.prod.yml, prometheus.yml, prometheus.dev.yml, prometheus.prod.yml, alerts/critical.yml, alerts/warnings.yml, alerts/info.yml, grafana/dashboards/*.json, scripts/smoke-test.sh, README.md

## 2. Remove ghost video metrics (6 files)

Remove all video-related alerts, dashboard panels, and documentation:
- `monitoring/prometheus/alerts/critical.yml` - Remove `VideoJobFailureSpike` alert
- `monitoring/prometheus/alerts/warnings.yml` - Remove `VideoJobsStuckPending` alert
- `monitoring/prometheus/alerts/info.yml` - Remove `VideoGenerationRateHigh` alert
- `monitoring/grafana/dashboards/background-jobs.json` - Remove "Video Status Check" panel
- `monitoring/scripts/smoke-test.sh` - Remove `video_requests_total` check
- `monitoring/README.md` - Remove "Video Generation Metrics" and "Video Generation Dashboard" sections

## 3. Fix Grafana port conflict (H5)

Change Grafana port mapping from `3000:3000` to `3001:3000` in `docker-compose.yml` to avoid collision with Next.js dev server. Update health check URL accordingly.

## 4. Fix Grafana default password (H6)

Replace hardcoded `GF_SECURITY_ADMIN_PASSWORD=admin` with `GF_SECURITY_ADMIN_PASSWORD=${GRAFANA_ADMIN_PASSWORD:-admin}` to allow override, and add a comment noting buyers should change this.

## 5. Remove --web.enable-lifecycle from prod Prometheus (M12)

Remove `--web.enable-lifecycle` from `docker-compose.prod.yml` to prevent unauthenticated shutdown via `/-/quit`.

## 6. Fix Prometheus metrics API key in scrape config (M17)

Add bearer_token or authorization header placeholder to `prometheus.prod.yml` scrape config, with a comment explaining the `METRICS_API_KEY` environment variable.

## 7. Update monitoring README

Rewrite `monitoring/README.md` to reflect DualStack branding and remove all video generation references.
