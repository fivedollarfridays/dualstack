# Objective

Harden the security posture across both stacks. Covers issues H1, H7, H8, M1, M2, M3.

# Implementation Plan

## 1. Add dev auth bypass warning (H1)

In `backend/app/main.py` lifespan, add a loud warning log when `clerk_jwks_url` is empty: "WARNING: Running without Clerk JWT validation. All X-User-ID headers are trusted. Do NOT deploy to production without setting CLERK_JWKS_URL." Also add this warning to .env.example as a comment.

## 2. Add security headers middleware (H7)

Create `backend/app/core/security_headers.py` with a middleware that adds to every response:
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy: camera=(), microphone=(), geolocation=()`

For the frontend, add `headers()` config to `next.config.mjs` with CSP, HSTS, X-Frame-Options.

## 3. Add basic rate limiting (H8)

Add `slowapi` to requirements.txt. Create a rate limiter in `backend/app/core/rate_limit.py`. Apply to billing checkout (10/min), item creation (30/min), and the webhook endpoint (60/min). Wire the existing `RateLimitError` to slowapi's exception.

## 4. Tighten CORS (M1)

In `backend/app/main.py`, replace `allow_methods=["*"]` with `allow_methods=["GET", "POST", "PATCH", "DELETE"]` and `allow_headers=["*"]` with `allow_headers=["Authorization", "Content-Type"]`.

## 5. Fix health endpoint error leak (M2)

In `backend/app/health/checks.py`, replace `error=str(e)` with `error="database unavailable"` and log the full exception server-side.

## 6. Add description max length (M3)

In `backend/app/items/schemas.py`, add `Field(max_length=10000)` to `description` in both `ItemCreate` and `ItemUpdate`.

## 7. Write tests for all changes

- Test security headers are present on responses
- Test rate limiting returns 429
- Test CORS rejects disallowed methods/headers
- Test health endpoint doesn't leak error details
- Test description max length validation
