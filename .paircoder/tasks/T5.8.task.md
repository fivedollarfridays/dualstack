# Objective

Add a GitHub Actions CI workflow that validates tests pass and builds work on every push and PR. Buyers expect a production-ready kit to include CI.

# Implementation Plan

## 1. Backend CI job

- Set up Python 3.13
- Install `requirements-dev.txt`
- Run `pytest --cov=app --cov-report=term-missing tests/`
- Fail if coverage drops below threshold

## 2. Frontend CI job

- Set up Node.js 18
- Run `npm ci`
- Run `npm run lint`
- Run `npm run test:coverage`
- Fail if coverage drops below threshold

## 3. Docker build job (optional validation)

- Build the backend Docker image (`docker build -t dualstack-api backend/`)
- Validates Dockerfile, .dockerignore, and requirements.txt are correct
- Does NOT push the image — build-only validation

### Files to create

- `.github/workflows/ci.yml`

# Acceptance Criteria

- [x] CI runs on push to main and on pull requests
- [x] Backend job: installs deps, runs pytest with coverage
- [x] Frontend job: installs deps, runs lint, runs jest with coverage
- [x] Docker build job: builds backend image successfully
- [x] All three jobs run in parallel
- [x] Workflow uses pinned action versions (e.g., actions/checkout@v4)
