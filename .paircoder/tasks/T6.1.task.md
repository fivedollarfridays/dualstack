# Objective

Wire the existing dev-mode auth components (ClerkAuthBridge, DevAuthProvider, AppUserButton, useAppAuth) into the actual application, and replace all direct `@clerk/nextjs` imports in pages/hooks with the abstracted `useAppAuth()` hook. Currently the frontend is non-functional without Clerk keys despite having a complete dev-mode auth system.

# Implementation Plan

## 1. Refactor `providers.tsx` to conditionally wire auth

- If `isClerkEnabled()` returns true: render `ClerkProvider` wrapping `ClerkAuthBridge`
- If `isClerkEnabled()` returns false: render `DevAuthProvider`
- Remove the `ClerkNotConfiguredError` component — dev mode handles this now
- Both paths provide `AuthContext` so `useAppAuth()` works everywhere

## 2. Replace direct `@clerk/nextjs` imports

- `frontend/src/app/(dashboard)/layout.tsx` — replace `UserButton` import with `AppUserButton`
- `frontend/src/app/(dashboard)/billing/page.tsx` — replace `useAuth` with `useAppAuth` from `@/contexts/auth-context`
- `frontend/src/app/(dashboard)/items/[id]/page.tsx` — replace `useAuth` with `useAppAuth`
- `frontend/src/hooks/use-items.ts` — replace `useAuth` with `useAppAuth` (4 hook functions)

## 3. Wire sign-in/sign-up pages

- `frontend/src/app/sign-in/[[...sign-in]]/page.tsx` — conditionally render `DevSignIn` or Clerk's `SignIn`
- `frontend/src/app/sign-up/[[...sign-up]]/page.tsx` — conditionally render `DevSignUp` or Clerk's `SignUp`

## 4. Update tests

- Update `providers.test.tsx` to test both Clerk and dev-mode paths
- Update affected test files to mock `useAppAuth` instead of Clerk
- Verify all 295 frontend tests still pass
