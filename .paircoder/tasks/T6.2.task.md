# Objective

Fix two critical backend security issues: (1) the environment field accepts any string, allowing auth bypass via typos like `ENVIRONMENT=staging`, and (2) Turso `libsql://` URLs are silently ignored, falling back to local SQLite without warning.

# Implementation Plan

## 1. Validate environment with Literal type

- Change `environment: str = "development"` to `environment: Literal["development", "production"] = "development"`
- Import `Literal` from `typing`
- Pydantic will reject invalid values at startup with a clear error message

## 2. Add Turso startup guard

- In `main.py` lifespan(), add a check: if `TURSO_DATABASE_URL` starts with `libsql://` and `ENVIRONMENT != "development"`, raise `RuntimeError` explaining that Turso integration requires `sqlalchemy-libsql` dialect
- In development, log a clear WARNING explaining the fallback to local SQLite
- Update the database.py fallback to include a prominent log message

## 3. Update tests

- Test that `Settings(environment="staging")` raises a Pydantic ValidationError
- Test that `Settings(environment="production")` works
- Test that `Settings(environment="development")` works
- Test Turso startup guard behavior in lifespan

### Files to modify

- `backend/app/core/config.py`
- `backend/app/main.py` (lifespan guard)
- `backend/app/core/database.py` (logging)
- `backend/tests/core/test_config.py` (or create)
- `backend/tests/test_main.py` (lifespan tests)
