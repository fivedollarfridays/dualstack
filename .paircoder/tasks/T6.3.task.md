# Objective

Fix the render.yaml Blueprint syntax (invalid `sync: false`) and add a database migration step so fresh deployments don't start with an empty database.

# Implementation Plan

## 1. Fix render.yaml syntax

- Remove `sync: false` from all env vars (invalid Render syntax)
- Env vars without a `value` field will prompt the user to set them in Render dashboard
- Add missing `CLERK_SECRET_KEY` env var entry
- Add `LOG_LEVEL` env var entry

## 2. Add migration to deployment

- Create an entrypoint script (`backend/entrypoint.sh`) that runs `alembic upgrade head` then starts uvicorn
- Update Dockerfile CMD to use the entrypoint script
- This ensures migrations run on every deploy

### Files to modify

- `render.yaml`
- `backend/Dockerfile`
- `backend/entrypoint.sh` (create)
