# Objective

Create a root-level `docker-compose.yml` for running the full application stack with one command, and fix monitoring stack defaults (Grafana password, alertmanager stubs, port binding).

# Implementation Plan

## 1. Create root `docker-compose.yml`

- Backend service: build from `backend/Dockerfile`, expose port 8000, env from `.env`
- Frontend service: build from a new `frontend/Dockerfile`, expose port 3000
- Both with health checks
- Reference `.env` file for configuration

## 2. Create `frontend/Dockerfile`

- Multi-stage build: install deps, build, run with `next start`
- Node 20 (LTS) base, non-root user, health check

## 3. Fix monitoring defaults

- **Grafana**: Remove `:-admin` fallback, require `GRAFANA_ADMIN_PASSWORD`
- **Alertmanager**: Replace `localhost:9999` stubs with commented-out Slack/PagerDuty templates
- **Port binding**: Change dev compose to `127.0.0.1:9090:9090` and `127.0.0.1:9093:9093`
- Pin monitoring images to specific versions (not `:latest`)

## 4. Update README quickstart

### Files to create/modify

- `docker-compose.yml` (create at project root)
- `frontend/Dockerfile` (create)
- `monitoring/docker-compose.yml` (modify)
- `monitoring/alertmanager/alertmanager.yml` (modify)
- `README.md` (update quickstart)
