# Objective

Fix non-deterministic pagination in the backend list query and the React anti-pattern of calling setState during render in the items page.

# Implementation Plan

## 1. Add ORDER BY to list_items query

- Add `.order_by(Item.created_at.desc())` to the select in `list_items()`
- Newest-first ordering is the expected default

## 2. Fix setState during render

- Wrap page-out-of-bounds reset in `useEffect` with `[data, page]` dependencies

## 3. Update tests

- Backend: Test items returned in created_at descending order
- Frontend: Verify page reset test still passes with useEffect

### Files to modify

- `backend/app/items/service.py`
- `backend/tests/items/test_service.py`
- `frontend/src/app/(dashboard)/items/page.tsx`
- `frontend/src/app/(dashboard)/items/items-page.test.tsx`
