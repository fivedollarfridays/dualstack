# Objective

Add a second layer of defense against dev-mode auth bypass (SEC-001) and remove the unnecessary `user_id` field from API responses (SEC-002).

SEC-001: `get_current_user_id` (auth.py:63-66) trusts the `X-User-ID` header whenever `clerk_jwks_url` is empty. The lifespan guard blocks this in `ENVIRONMENT=production`, but if a deploy uses `ENVIRONMENT=development` with a public-facing URL, any caller can impersonate any user. Add a defense-in-depth check inside `get_current_user_id` itself.

SEC-002: `ItemResponse` (items/schemas.py:33) returns the internal Clerk `user_id` to the client. Since items are already scoped to the authenticated user, this leaks an internal identifier unnecessarily.
