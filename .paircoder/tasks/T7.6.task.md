# Objective

Add a structured audit log for write operations (SEC-013, SOC2 gap) and add rate limits to item read/update/delete endpoints (SEC-014). Currently only `create_item` has a rate limit.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/core/audit.py` | New file: structured audit logger for write operations |
| `backend/app/items/routes.py` | Add rate limits to all item endpoints; add audit logging to write endpoints |
| `backend/tests/core/test_audit.py` | Tests for audit logging |
| `backend/tests/items/test_routes.py` | Verify rate limits exist |

# Implementation Plan

1. Write failing tests for `log_audit_event` structured logging
2. Create `audit.py` with structlog-based audit event logger
3. Write failing tests for rate limits on list/get/update/delete
4. Add rate limits: 60/min for reads, 30/min for writes
5. Wire audit logging into create/update/delete routes

# Acceptance Criteria

- [x] All item CRUD endpoints have rate limits
- [x] Write operations (create, update, delete) emit structured audit log events
- [x] Audit log includes user_id, action, resource_type, resource_id, timestamp

- [x] All tests pass
