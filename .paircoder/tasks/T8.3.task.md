# Objective

Reduce information disclosure on health endpoints (NEW-005) and remove the IDOR-prone `customer_id` field from `PortalRequest` (NEW-007).

# Files to Update

| File | Change |
|------|--------|
| `backend/app/health/models.py` | Remove `uptime_seconds` from `LivenessStatus`, remove `latency_ms` from `ServiceCheck` public response |
| `backend/app/health/checks.py` | Simplify `/health/live` to return `{"alive": true}` only; remove `latency_ms` from `/health/ready` checks |
| `backend/app/billing/schemas.py` | Remove `customer_id` from `PortalRequest` |
| `backend/tests/health/test_checks.py` | Update tests for simplified responses |
| `backend/tests/billing/test_schemas.py` | Update tests for PortalRequest without customer_id |

# Implementation Plan

1. Remove `uptime_seconds` from `LivenessStatus` model; simplify `/health/live` to `{"alive": true}`
2. Remove `latency_ms` from `ServiceCheck` model (or keep internal but exclude from response); update `/health/ready` accordingly
3. Remove `customer_id` from `PortalRequest` schema; add comment that customer ID must be resolved server-side from `user_id`
4. Update tests to match

# Acceptance Criteria

- [x] `GET /health/live` returns `{"alive": true}` without `uptime_seconds`
- [x] `GET /health/ready` does not include `latency_ms` in response
- [x] `PortalRequest` schema does not accept `customer_id` from client input
- [x] All tests pass
- [x] Architecture checks pass
