# Objective

Expand structured audit logging to cover auth failures, data read operations, and billing events (NEW-006). Addresses SOC2 CC6.1 and CC7.1 gaps.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/core/audit.py` | Add optional `outcome` and `detail` parameters to `log_audit_event` |
| `backend/app/core/auth.py` | Add `log_audit_event` calls on auth failures (invalid JWT, missing header, dev-mode rejection) |
| `backend/app/items/routes.py` | Add `log_audit_event` calls for `list_items_route` and `get_item_route` (action="read") |
| `backend/app/billing/routes.py` | Add `log_audit_event` call for `create_checkout` (action="billing.checkout") |
| `backend/tests/core/test_audit.py` | Add tests for outcome/detail parameters |
| `backend/tests/core/test_auth.py` | Verify auth failures emit audit events |
| `backend/tests/items/test_routes.py` | Verify read operations emit audit events |

# Implementation Plan

1. Extend `log_audit_event` signature to accept optional `outcome: str = "success"` and `detail: str = ""` parameters
2. In `auth.py`, add `log_audit_event` calls in each `raise AuthenticationError` path (outcome="failure")
3. In `items/routes.py`, add `log_audit_event` calls to `list_items_route` (action="list") and `get_item_route` (action="read")
4. In `billing/routes.py`, add `log_audit_event` call to `create_checkout` route (action="billing.checkout")
5. Write tests verifying each new audit call

# Acceptance Criteria
