# Objective

Harden the Stripe webhook handler to reject requests when `stripe_webhook_secret` is not configured, and add audit logging for successfully processed webhook events to satisfy SOC2 CC7.1 requirements.

# Findings Addressed

- **AUDIT-001** (Critical): Stripe webhook accepts requests when `stripe_webhook_secret` is empty in dev; handler is a stub that does nothing
- **AUDIT-008** (Medium): Successful webhook events not audit-logged (SOC2 CC7.1 gap)

# Files to Update

| File | Change |
|------|--------|
| `app/billing/service.py` | Guard `handle_webhook` when secret is empty; add audit logging for successful events |
| `app/billing/routes.py` | Return 503 if webhook secret not configured |
| `tests/billing/test_service.py` | Tests for empty-secret rejection and audit logging |
| `tests/billing/test_routes.py` | Route-level test for 503 when unconfigured |

# Implementation Plan

1. In `handle_webhook`, check if `stripe_webhook_secret` is empty before attempting signature verification; raise a clear error or return early
2. In the webhook route, catch the unconfigured state and return 503 Service Unavailable
3. Add `log_audit_event` calls for `checkout.session.completed` and `customer.subscription.updated` events
4. Write TDD tests for all paths

# Acceptance Criteria

- [x] Webhook returns 503 when `stripe_webhook_secret` is empty
- [x] Successful `checkout.session.completed` events are audit-logged with `outcome="success"`
- [x] Successful `customer.subscription.updated` events are audit-logged with `outcome="success"`
- [x] Unhandled event types are not audit-logged (no noise)
- [x] All existing webhook tests continue to pass
- [x] New tests cover empty-secret and audit-logging paths
