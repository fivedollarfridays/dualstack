# Objective

Fix infrastructure security issues: scope proxy IP forwarding, enable Prometheus auth in prod, and add authentication between Grafana and Prometheus.

# Findings Addressed

- **AUDIT-002** (High): `--forwarded-allow-ips='*'` enables IP spoofing for rate limiting outside Render
- **AUDIT-004** (High): Prometheus prod config has auth section commented out by default
- **AUDIT-014** (Low): No auth between Grafana and Prometheus on shared Docker network

# Files to Update

| File | Change |
|------|--------|
| `backend/entrypoint.sh` | Make `--forwarded-allow-ips` configurable via env var, default to restrictive |
| `monitoring/prometheus/prometheus.prod.yml` | Uncomment and configure basic_auth section |
| `monitoring/prometheus/web.yml` | Create web config with basic_auth for Prometheus |
| `monitoring/grafana/provisioning/datasources/datasource.yml` | Add basicAuth credentials for Prometheus datasource |
| `monitoring/docker-compose.prod.yml` | Mount web.yml into Prometheus container |

# Implementation Plan

1. Replace `--forwarded-allow-ips='*'` with `--forwarded-allow-ips="${FORWARDED_ALLOW_IPS:-127.0.0.1}"` in entrypoint.sh
2. Create `monitoring/prometheus/web.yml` with basic_auth configuration using env-substituted credentials
3. Uncomment auth section in prometheus.prod.yml and reference web.yml
4. Update Grafana datasource provisioning to include basicAuth for Prometheus
5. Document the required environment variables

# Acceptance Criteria

- [x] `--forwarded-allow-ips` reads from `FORWARDED_ALLOW_IPS` env var with safe default
- [x] Prometheus prod config has basic_auth enabled (not commented out)
- [x] `web.yml` exists with basic_auth configuration
- [x] Grafana datasource includes basicAuth credentials for Prometheus
- [x] Documentation notes required env vars for monitoring auth
