# Objective

Clean up misplaced secrets in .env.example files and add startup warnings for missing Stripe configuration.

# Findings Addressed

- **AUDIT-003** (High): Frontend .env.example contains server-side secrets (CLERK_SECRET_KEY, STRIPE_SECRET_KEY)
- **AUDIT-009** (Medium): CLERK_SECRET_KEY in backend .env.example but never loaded by Settings
- **AUDIT-016** (Info): No warning when stripe_secret_key is empty in dev

# Files to Update

| File | Change |
|------|--------|
| `frontend/.env.example` | Remove CLERK_SECRET_KEY, STRIPE_SECRET_KEY; keep only client-side vars |
| `backend/.env.example` | Remove CLERK_SECRET_KEY (not used by Settings model) |
| `backend/app/main.py` | Add startup warning when stripe_secret_key is empty in non-production |

# Implementation Plan

1. Edit frontend/.env.example to remove server-side secrets, add comments explaining which vars are client-side only
2. Edit backend/.env.example to remove CLERK_SECRET_KEY (Settings model doesn't load it)
3. Add a startup log warning in create_app() when stripe_secret_key is empty and environment != production

# Acceptance Criteria

- [x] frontend/.env.example contains NO server-side secrets (no CLERK_SECRET_KEY, STRIPE_SECRET_KEY)
- [x] backend/.env.example does NOT contain CLERK_SECRET_KEY
- [x] Backend logs a warning at startup when stripe_secret_key is empty in dev/staging
- [x] No production behavior changes
