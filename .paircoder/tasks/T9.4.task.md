# Objective

Add Content-Security-Policy headers to the Next.js frontend and validate redirect URLs before navigating to Stripe checkout/portal.

# Findings Addressed

- **AUDIT-005** (High): Frontend redirects to backend-controlled URL via `window.location.href` without validating it's a Stripe domain
- **AUDIT-007** (Medium): No CSP header on frontend (Next.js) -- primary XSS mitigation missing

# Files to Update

| File | Change |
|------|--------|
| `frontend/next.config.mjs` | Add security headers including CSP |
| `frontend/src/app/billing/page.tsx` | Validate redirect URL is a Stripe domain before navigation |
| `frontend/src/lib/url-validation.ts` | Create URL validation utility for allowed redirect domains |
| `frontend/src/lib/url-validation.test.ts` | Tests for URL validation |

# Implementation Plan

1. Create `url-validation.ts` with a function that validates URLs against an allowlist of domains (checkout.stripe.com, billing.stripe.com)
2. Update billing page to validate the checkout/portal URL before `window.location.href` assignment
3. Add CSP and other security headers to `next.config.mjs` via the `headers()` config
4. Write tests for URL validation utility

# Acceptance Criteria

- [x] `window.location.href` only navigates to validated Stripe domains
- [x] Invalid redirect URLs throw an error / show user-facing message instead of navigating
- [x] Next.js serves CSP header on all pages
- [x] CSP allows Clerk, Stripe, and self resources
- [x] URL validation has unit tests covering valid/invalid/malicious URLs
- [x] Existing frontend tests pass
