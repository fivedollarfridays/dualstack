# Objective

Sanitize error responses, remove unnecessary security headers from API responses, and tighten CORS credentials handling in dev mode.

# Findings Addressed

- **AUDIT-006** (Medium): NotFoundError echoes client-supplied `item_id` in error response
- **AUDIT-012** (Low): Backend CSP header meaningless on JSON API responses
- **AUDIT-013** (Low): `allow_credentials=True` with dev-mode header trust creates subtle CSRF-adjacent risk

# Files to Update

| File | Change |
|------|--------|
| `app/items/service.py` | Sanitize item_id before including in error messages |
| `app/core/security_headers.py` | Skip CSP header for JSON API responses (Content-Type check) |
| `app/main.py` | Set `allow_credentials=False` when using dev-mode X-User-ID auth |
| `tests/items/test_service.py` | Test that error messages don't echo raw input |
| `tests/core/test_security_headers.py` | Test conditional CSP behavior |
| `tests/test_main.py` | Test CORS credentials behavior |

# Implementation Plan

1. In items service, validate/sanitize item_id format before including in NotFoundError message (e.g., truncate, strip special chars)
2. In SecurityHeadersMiddleware, skip CSP header when response Content-Type is application/json
3. In create_app(), set `allow_credentials=False` when `clerk_jwks_url` is not set (dev mode)
4. Write TDD tests for all changes

# Acceptance Criteria

- [x] NotFoundError messages do not echo raw client input (sanitized or generic message)
- [x] JSON API responses do not include CSP header
- [x] HTML responses (if any) still include CSP header
- [x] Dev-mode CORS does not set `allow_credentials=True`
- [x] All existing tests pass
- [x] New tests cover sanitization, conditional CSP, and CORS credential behavior
